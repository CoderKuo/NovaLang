# 宿主绑定配置规范

本文档定义工作区 `.nova/*.json` 的宿主绑定（host bindings）格式，以及 VS Code 对 YAML 内嵌 Nova 脚本的识别与命名空间规则。

## 1. 目的

宿主绑定配置用于描述“宿主应用在运行时会向脚本上下文注入哪些变量、函数和对象”。

该配置主要供编辑器/LSP 使用，用于：

- 代码补全
- 悬停提示
- 签名帮助
- YAML 内嵌 Nova 脚本的命名空间隔离

## 2. 加载规则

VS Code 扩展会扫描当前工作区根目录下的：

- `.nova/*.json`

加载行为：

- 只扫描 `.nova` 目录下的 `.json` 文件
- 按文件名字典序排序后依次加载
- 后加载的同名符号会覆盖前面同名符号
- 若存在 `default` 命名空间，其他命名空间会默认继承 `default`

## 3. 顶层结构

最小结构如下：

```json
{
  "version": 1,
  "globals": [],
  "namespaces": {}
}
```

字段说明：

- `version`：当前版本，固定为 `1`
- `globals`：全局宿主绑定数组，可选
- `namespaces`：命名空间对象，可选

## 4. 命名空间结构

```json
{
  "namespaces": {
    "default": {
      "globals": []
    },
    "reward": {
      "extends": ["default"],
      "globals": []
    }
  }
}
```

字段说明：

- `extends`：命名空间继承列表，可选，必须是字符串数组
- `globals`：该命名空间下的宿主符号数组，可选

推荐约定：

- `default`：默认命名空间
- 业务命名空间：如 `condition`、`reward`、`ui`

## 5. 符号结构

### 5.1 变量 / 属性

```json
{
  "name": "player",
  "kind": "variable",
  "type": "Player",
  "mutable": false,
  "documentation": "当前玩家对象"
}
```

支持字段：

- `name`：必填，符号名
- `kind`：必填，`variable` 或 `property`
- `type`：可选，类型名
- `mutable`：可选，是否可变，默认 `false`
- `documentation`：可选，文档说明

### 5.2 函数

```json
{
  "name": "giveItem",
  "kind": "function",
  "parameters": [
    { "name": "itemId", "type": "String" },
    { "name": "count", "type": "Int" }
  ],
  "returnType": "Unit",
  "documentation": "发放奖励"
}
```

支持字段：

- `name`：必填
- `kind`：必填，固定为 `function`
- `parameters`：可选，参数数组
- `returnType`：可选，返回类型
- `documentation`：可选，文档说明

### 5.3 对象

```json
{
  "name": "server",
  "kind": "object",
  "documentation": "宿主服务对象",
  "members": [
    {
      "name": "broadcast",
      "kind": "function",
      "parameters": [
        { "name": "message", "type": "String" }
      ],
      "returnType": "Unit"
    },
    {
      "name": "onlineCount",
      "kind": "property",
      "type": "Int"
    }
  ]
}
```

支持字段：

- `name`：必填
- `kind`：必填，固定为 `object`
- `documentation`：可选
- `members`：可选，成员数组

## 6. 参数结构

```json
{
  "name": "message",
  "type": "String"
}
```

字段说明：

- `name`：必填
- `type`：可选

## 7. 支持的 kind

当前支持：

- `variable`
- `function`
- `object`
- `property`

## 8. YAML 内嵌 Nova 标记规则

当前支持在 YAML 文本块中使用以下标记：

### 8.1 默认命名空间

```yaml
# nova
script: |-
  log("hello")
```

等价于：

- `namespace = default`

### 8.2 显式命名空间

```yaml
# nova=reward
script: |-
  giveItem("diamond", 3)
```

或：

```yaml
script: |- # nova=reward
  giveItem("diamond", 3)
```

## 9. 诊断规则

### 9.1 命名空间不存在

当 YAML 中写了：

```yaml
# nova=reward
```

但 `.nova/*.json` 中没有 `reward` 时，扩展会在 YAML 标记行给出 warning：

- `Unknown Nova namespace 'reward'.`

### 9.2 JSON 结构错误

扩展会对 `.nova/*.json` 做基础校验，包括：

- `version` 必须为 `1`
- `globals` 必须是数组
- `namespaces` 必须是对象
- `extends` 必须是字符串数组
- `name` 必须是非空字符串
- `kind` 必须是 `variable/function/object/property`
- `parameters` / `members` 必须是数组

校验错误会直接显示在对应 JSON 文件中。

## 10. 完整示例

```json
{
  "version": 1,
  "globals": [
    {
      "name": "log",
      "kind": "function",
      "parameters": [
        { "name": "message", "type": "String" }
      ],
      "returnType": "Unit",
      "documentation": "输出日志"
    }
  ],
  "namespaces": {
    "default": {
      "globals": [
        {
          "name": "player",
          "kind": "variable",
          "type": "Player",
          "documentation": "当前玩家对象"
        }
      ]
    },
    "reward": {
      "extends": ["default"],
      "globals": [
        {
          "name": "giveItem",
          "kind": "function",
          "parameters": [
            { "name": "itemId", "type": "String" },
            { "name": "count", "type": "Int" }
          ],
          "returnType": "Unit",
          "documentation": "发放奖励"
        }
      ]
    }
  }
}
```

## 11. 推荐实践

- 把公共注入项放进 `default`
- 用 `reward` / `condition` / `ui` 这种业务名区分脚本块能力域
- 优先保持宿主绑定“显式声明”，不要依赖运行时反射结果作为唯一来源
- 大型项目可以拆成多个 `.nova/*.json` 文件按模块组织
